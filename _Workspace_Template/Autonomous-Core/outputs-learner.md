---

## Execution Logic

```
ON Phase8_Complete (quality_score >= 90):
    
    1. READ Outputs/[topic]/manifest.json
       - Get quality_scores
       - Get topic/domain
       - Get file list
    
    2. READ Outputs/[topic]/article.md
       - Extract structure patterns
       - Extract voice patterns
       - Extract E-E-A-T signals
    
    3. ANALYZE patterns:
       - Compare with existing EXPERIENCE files
       - Identify unique patterns
       - Calculate pattern novelty score
    
    4. IF novelty_score >= 0.7:
       a. GENERATE new EXPERIENCE file:
          - EXP-[next_id]-[topic-slug].md
       b. STRUCTURE:
          - Pattern summary
          - Reusable templates
          - Success metrics
          - Example snippets
       c. WRITE to Knowledge-Base/EXPERIENCE/
       d. UPDATE knowledge-base-index.json
    
    5. IF novelty_score < 0.7:
       a. MERGE patterns into existing EXP file
       b. UPDATE success metrics
       c. INCREMENT usage count
    
    6. LOG to Memory/Learning/experience-log.md
```

## Pattern Categories for Experience

| Category | What to Extract |
|----------|-----------------|
| **Structure** | H1/H2/H3 hierarchy, section flow, TL;DR format |
| **SEO** | Keyword placement, meta patterns, URL slugs |
| **E-E-A-T** | Expert citations, trust signals, authority format |
| **AEO** | Featured snippet format, PAA patterns |
| **Voice** | Tone patterns, active voice, engagement style |
| **Domain** | Industry-specific terminology, local signals |

## Quality Gates

| Gate | Threshold | Action |
|------|-----------|--------|
| Quality Score | ≥ 90 | Proceed to extraction |
| Novelty Score | ≥ 0.7 | Create new EXP file |
| Novelty Score | < 0.7 | Merge into existing |
| Pattern Count | ≥ 3 | Required for new EXP |
